GitHub Pages简介

每个GitHub用户都可开通一个属于自己的静态网站，域名为username.github.io。这个网站与一个托管在GitHub上的代码仓库相关联，用户可以像更新普通代码仓库一样更新自己的静态博客网站。

 

Hexo简介

Hexo（https://hexo.io/zh-cn/）是一个基于node.js的博客框架，支持利用Markdown编写博文，用户可以用一条指令轻松将Hexo部署到自己的GitHub page上。此外，Hexo还可以与多种模板引擎（EJS，Pug，Nunjucks）和工具（Babel，PostCSS，Less/Sass）轻易集成。

 

 作为一个博客搭建的入门教程，本文只介绍以上工具最基本的使用方法，更多内容请读者自行探索。

第三步：安装Hexo

■安装Hexo需要先安装Git和Node.js，由于本文篇幅有限，软件安装也相对简单，在此只提供教程连接

（https://git-scm.com/book/zh/v2/起步-安装-Git、https://nodejs.org/en/）

 

■使用命令 npm install -g hexo-cli安装Hexo

■Hexo安装结束后，执行以下命令初始化博客文件夹：

$ hexo init $ cd $ npm install

建立完成后，应有如下目录

├── _config.yml ├── package.json ├── scaffolds ├── source | └── _posts └── themes

 

其中，config.yml是BLOG的配置文件，其具体内容及含义详见此处；

https://hexo.io/zh-cn/docs/configuration

scaffolds是Hexo的模板文件夹；新建文章时Hexo会根据模板内容默认填充至文章中；source文件夹存放用户文章和草稿，themes文件夹存放博客主题。

第五步：生成静态文件并部署至GitHub Page

■执行 hexo g 命令生成静态文件 Git新用户需要配置SSH，详见此处

https://help.github.com/en/github/authenticating-to-github/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent

■输入 npm install hexo-deployer-git –save 安装hexo-deployer-git

■修改_config.yml文件中的deploy部分，type字段赋值为git，repo字段赋值为GitHub上新建仓库的SSH链接，如图所示

■执行hexo d 命令部署到Github

■访问https://username.github.io/即可访问部署好的博客
